# 切片：学习看板

## 目标

用户可以在学习 Tab 查看已加入课程和学习概览。真实“继续学习”依赖后续最近学习位置能力；本切片只保留看板中的入口占位，不把它伪装成跳转。

## 用户流程

1. 用户打开学习 Tab。
2. App 调用 `GET /api/frontend/dashboard`。
3. App 渲染已加入课程、教材数量和累计学习时长。
4. 如果没有已加入课程，用户可从空状态跳转到课程 Tab。
5. 如果已有加入课程，课程卡片展示“继续学习”入口占位，但不实现最近学习跳转。

## API

- `GET /api/frontend/dashboard`

## 页面

- `DashboardScreen`

## 组件

- `AppCard`
- `AppButton`
- `StateView`

## 状态

- loading
- empty
- error
- normal

## 不做范围

- 课程详情
- 教材列表和阅读
- 真实阅读进度
- 最近学习位置记录与跳转
- AI 问答
- 教材上传、提炼和向量化

## 验收标准

- 学习看板通过 `frontendService` 加载。
- 网络失败时显示可重试错误状态。
- 没有已加入课程时显示空状态，并可跳转到课程 Tab。
- 有已加入课程时显示课程数、教材数、学习时长和课程卡片。
- “继续学习”不跳转到课程 Tab 冒充进度恢复；最近学习位置能力就绪前仅作为禁用或占位入口展示。
- Screen 层没有直接 `fetch`。
