# 切片：Beta 打磨

## 目标

在学习闭环和基础 AI 问答可用后，集中处理错误体验、缓存、阅读可读性、UI/UX 打磨和 Android 真机稳定性，让 Beta 版本适合连续使用。0.9.0 可以基于 AI 设计图进行所见即所得式的界面重做和交互优化。

## 用户流程

1. 用户在课程、教材、AI 问答和设置等主路径中连续操作。
2. App 在网络波动、内容缺失、缓存命中和重试场景下保持可理解状态。
3. 用户阅读 `bookinfo` / `bookdetail` 时看到更适合阅读的文本，而不是裸 XML 标签。
4. Android 真机验证主路径可打开、可返回、可重试、不卡死。

## 页面

- 学习看板
- 课程列表与课程详情
- 教材详情与教材阅读
- AI 问答
- 设置页

## 组件 / 工具

- `StateView`
- `BookContentSection`
- 基于 AI 设计图落地的页面级 UI/UX 组件调整
- 轻量 XML 阅读格式化工具

## 状态

- loading
- empty
- error
- retrying
- cached
- normal

## 不做范围

- 重写 `bookinfo` / `bookdetail` 后端结构
- 完整 XML 结构化解析器
- 为了打磨体验而引入大型第三方 UI 框架；优先在现有 React Native / Expo / 本地 design 组件基础上改造
- iOS 上架流程

## 验收标准

- 主路径错误状态文案清晰，并提供可用的重试入口。
- 可以根据 AI 设计图对主路径页面做大幅 UI/UX 调整，但不破坏既有服务层和导航边界。
- 常见数据可缓存或复用，返回页面时不出现明显空白闪烁。
- `bookinfo` / `bookdetail` 阅读页不直接暴露裸 XML 标签；展示层可通过轻量工具去掉标签、保留文本、处理常见实体，并按标题/段落插入换行。
- XML 阅读格式化失败时保留原始内容展示，不阻塞阅读页。
- Android 真机完成学习看板、课程、教材阅读、AI 问答、设置页基础路径验证。
- TypeScript 检查通过。
